# 파워유저 프롬프트 패턴

> Claude를 깊이 활용하는 사용자들이 검증한 프롬프트 기법 모음.
> 각 패턴은 독립적으로 또는 조합해서 사용 가능.

---

## 1. Chain-of-Thought (CoT) — 단계별 추론 강제

```
이 문제를 단계별로 생각해보세요.

문제: {문제}

단계:
1. 먼저 핵심 요소를 파악하고
2. 각 선택지의 트레이드오프를 나열한 뒤
3. 최선의 접근 방식을 결정하세요.

결론을 내리기 전에 각 단계를 명시적으로 작성하세요.
```

**효과**: 복잡한 구현·디버깅 결정 품질이 눈에 띄게 향상됨.

---

## 2. Few-Shot — 예시로 형식 고정

```
다음 형식으로 변환하세요.

입력: "PM2 재시작했더니 프로세스가 안 뜸"
출력: {"category": "운영", "severity": "high", "action": "pm2 logs로 에러 확인 후 ecosystem.config.cjs 점검"}

입력: "텔레그램 봇 응답 없음"
출력: {"category": "통신", "severity": "critical", "action": "봇 토큰 확인 → 채널 세션 재시작"}

입력: "{실제 입력}"
출력:
```

**효과**: 출력 형식을 100% 고정. JSON 파싱 실패율 제거.

---

## 3. 역할 + 제약 분리 패턴

```
역할: 당신은 시니어 DevOps 엔지니어입니다.
전문 분야: Windows 서버, PM2, Node.js, PowerShell

제약:
- 코드는 반드시 실행 가능한 완성본으로 제공
- 설명보다 코드 우선
- Windows 경로 구분자(\\) 사용

작업: {작업 내용}
```

**효과**: 역할과 제약을 동시에 선언하면 불필요한 설명이 줄고 바로 쓸 수 있는 코드가 나옴.

---

## 4. Self-Critique — 자기 검토 루프

```
{작업} 을 수행한 후, 다음을 검토하세요:

검토 항목:
- 엣지 케이스를 놓치지 않았는가?
- 더 단순한 방법이 있는가?
- 보안 취약점이 없는가?

검토 후 문제가 있으면 수정하고, 없으면 "검토 완료"라고 명시하세요.
```

**효과**: 한 번의 프롬프트로 구현 + 코드리뷰 효과.

---

## 5. 출력 형식 강제 패턴

```
응답은 반드시 다음 구조로만 작성하세요:

## 결론
(1~2줄, 핵심 결과만)

## 코드
```코드블록```

## 주의사항
- (있을 경우에만 기재)

다른 형식이나 서론·결론 문장 없이 위 구조만 사용하세요.
```

**효과**: 모바일에서 스크롤 없이 핵심을 바로 파악. 하네스에서 특히 유용.

---

## 6. 컨텍스트 압축 전달 패턴

```
현재 상황:
- 프로젝트: {프로젝트명}
- 파일: {파일 경로}
- 에러: {에러 메시지 1줄}
- 마지막 시도: {마지막으로 한 것}

질문: {핵심 질문만}
```

**효과**: 긴 배경 설명 없이 Claude가 바로 핵심에 집중. 토큰 절약 + 응답 품질 향상.

---

## 7. 점진적 상세화 (Progressive Disclosure)

```
[L1] 30자 이내 핵심 결과
[L2] 3~5줄 요약 (방법 + 결과)
[L3] 전체 상세 (코드·로그·원인)

L1만 먼저 보내고, 사용자가 "자세히"라고 하면 L2, "전체 로그"라고 하면 L3를 보내세요.
```

**효과**: 모바일 알림 공간 최적화. 필요할 때만 상세 정보 요청 가능.

---

## 8. 메모리 주입 패턴 (캐시 최적화)

```python
# Anthropic SDK — cache_control 활용
system = [
    {
        "type": "text",
        "text": "당신은 {역할}입니다.\n\n## 사용자 메모리\n{memory_content}",
        "cache_control": {"type": "ephemeral"}  # 5분 캐시
    }
]
```

**효과**: 긴 메모리/컨텍스트를 매번 전송하지 않고 캐시. 토큰 비용 60~80% 절감.

---

## 9. 작업 경계 명확화 패턴

```
이 작업의 범위:
✅ 할 것: {구체적으로 할 것}
❌ 하지 말 것: {건드리지 말 것}
⚠️ 불확실하면: 실행 전에 먼저 확인

시작 전에 이 범위를 한 줄로 확인해주세요.
```

**효과**: 범위를 벗어난 수정 방지. 위험 작업에서 특히 유효.

---

## 10. 멀티 에이전트 역할 분리 패턴

```
이 작업을 두 단계로 처리하세요:

[플래너] 먼저 실행 계획을 JSON으로 작성:
{
  "steps": ["단계1", "단계2"],
  "risks": ["위험요소"],
  "validation": ["검증 기준"]
}

[실행자] 플래너의 계획을 그대로 실행하고 각 단계 완료 시 체크:
✅ 단계1 완료
✅ 단계2 완료
```

**효과**: 계획 없이 바로 실행하는 것보다 오류율이 크게 낮아짐. 하네스 5-역할의 핵심.

## 조합 예시

**디버깅 시**:
`컨텍스트 압축` + `CoT` + `Self-Critique`

**구현 시**:
`역할+제약` + `작업 경계` + `멀티 에이전트`

**모바일 운영 시**:
`점진적 상세화` + `출력 형식 강제` + `메모리 주입`
